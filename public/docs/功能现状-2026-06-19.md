# 城市足迹地图 · 功能现状文档

本文档按 2026-07-16 当前代码与生产展示策略整理。项目不是 SaaS，而是一个用于记录个人回忆的城市足迹应用；工程上保留可维护、可扩展的结构，界面上优先呈现日常记录真正会用到的内容。

## 一、产品定位

城市足迹地图用于记录去过的中国城市、停留天数、最后停留日期和备注，并在地图上点亮城市。它面向个人回忆和少量好友参与记录，不包含付费、团队协作、商业运营、复杂权限审批等 SaaS 功能。

公开界面重点保留：

- 城市地图与省市下钻；
- 城市访问记录的添加、编辑、删除；
- 足迹统计面板、排行和成就徽章；
- 个人资料与主题切换；
- 管理员基础维护；
- 系统升级记录。

公开界面收起：

- 普通用户访问记录导入、导出、模板下载；
- 管理员数据管理导入、导出、模板下载；
- 内部维护文档入口。

这些维护能力仍可作为本地工程能力保留在代码中，但不作为公开界面的默认入口。

## 二、技术栈与运行结构

- 前端：React 18、TypeScript、Vite、Zustand、ECharts、自建 CSS token 样式系统。
- 后端：FastAPI、SQLite、JWT、SlowAPI。
- API：前端统一请求 `/cityprint/api/*`，登录后通过 `Authorization: Bearer <token>` 鉴权。
- 样式：Tailwind 已移除，当前使用 `src/index.css` 中的设计变量、组件 class 和主题变量。
- 共享组件：`src/components/ui` 指向共享 UI 组件库，当前主要使用 `DateInput` 和 `FuzzySelect`。
- 展示模式开关：生产环境通过配置收起导入/导出类维护入口，并避免拆出不需要的 `xlsx` 运行 chunk。

## 三、角色与权限

未登录用户只能看到登录/注册流程。

普通用户可以：

- 登录、注册；
- 查看地图；
- 管理自己的访问记录；
- 查看自己的统计、排行和成就；
- 修改昵称、修改密码；
- 切换主题。

管理员可以额外：

- 打开系统管理面板；
- 创建普通用户；
- 修改用户昵称；
- 重置普通用户密码；
- 删除普通用户；
- 查看全站访问记录和系统统计；
- 查看系统文档与升级记录。

后端管理员接口均有权限保护。普通访问记录接口始终按当前 token 所属用户隔离。

## 四、核心功能

### 4.1 登录与启动

应用启动时会恢复本地 token；如果 token 有效，会拉取当前用户、访问记录、成就和主题设置。登录成功后先显示应用框架，再延迟加载地图资源，避免首次进入时长时间空白。

登录页提供用户登录、管理员登录和普通用户注册。管理员身份由后端返回的 `is_admin` 判断，普通注册不会创建管理员。

### 4.2 顶部栏与主题

顶部栏展示品牌、当前用户和账户菜单。账户菜单提供个人资料、主题选择、系统管理和退出登录。主题当前支持：

- 樱花；
- 晨雾；
- 琥珀；
- 绿松；
- 冰河。

主题通过 CSS 变量切换，地图点亮色、按钮、表格、弹窗和滚动条都应跟随主题适配。

### 4.3 地图主视图

地图使用 ECharts 渲染中国城市与省界。全国视图展示城市点亮状态，点击城市会进入省级视图或打开城市详情。省级视图中点击城市会打开城市详情抽屉。

地图支持：

- 全国/省级视图切换；
- 桌面端滚轮缩放、拖拽平移；
- 移动端触摸平移、双指缩放；
- 重置视图；
- 放大、缩小；
- 按停留时间或最后离开时间切换染色模式。

地图工具条在桌面端和移动端都采用轻量图标按钮，放在左侧地图控制组内，避免遮挡右侧足迹统计面板。

### 4.4 城市详情与访问记录

城市详情抽屉展示城市名、省份、大区、累计停留天数和已有记录。每条记录包含：

- 停留天数；
- 最后停留日期；
- 备注。

用户可以添加、编辑、删除自己的访问记录。同一城市允许多条记录，用于记录不同时间段或不同回忆片段。

### 4.5 足迹统计面板

足迹统计是公开界面保留的核心功能。面板展示：

- 点亮城市数；
- 访问次数；
- 覆盖省份；
- 累计天数；
- 城市覆盖率；
- 成就徽章；
- 城市排行。

统计数字可点击进入个人资料中的访问记录视图。面板支持收起和展开；桌面端固定在右侧，移动端作为页面下方统计区展示。

### 4.6 成就系统

成就由前端根据访问记录计算，并写入后端持久化。当前包括初次点亮、十城达人、省份征服者、百城俱乐部、长居达人、神州北极、东极迎光、南海逐浪、西域极境、全国制霸等成就。

未解锁成就以低强调样式展示，已解锁成就突出显示。

### 4.7 个人资料

个人资料包含：

- 用户名；
- 昵称修改；
- 密码修改；
- 访问记录表格；
- 主题设置。

用户名为账号标识，不可在个人资料中修改。昵称和密码在只读态、编辑态和保存态使用统一的输入框与按钮规范。

访问记录页保留添加、编辑、删除、清空、统计入口；生产展示模式收起导入、导出和模板下载。

### 4.8 管理员面板

管理员面板包含三个页签：

- 用户管理；
- 数据管理；
- 系统文档。

用户管理支持新增普通用户、修改昵称、重置密码和删除普通用户。

数据管理用于查看全站访问记录，并支持按用户名、昵称、省份、城市筛选。生产展示模式收起数据导入、导出和模板下载入口，保留查询和表格查看能力。

系统文档保留：

- 设计系统文档；
- 功能现状文档；
- 系统升级记录。

内部维护文档不在公开系统文档中展示。

## 五、已知保留能力与技术债

以下能力或代码当前保留，但不作为公开界面主要入口：

- Excel 导入、导出和模板下载：本地项目可继续保留，生产展示模式默认收起；
- `PosterGenerator`：海报生成组件存在，但当前无入口；
- `SearchDropdown`：搜索组件存在，但当前无入口；
- `SettingsPanel`：旧设置弹窗存在，但当前实际设置入口在个人资料内；
- `DrillDownStats` 独立弹窗模式：组件支持，但当前主要以内嵌统计方式使用。

后续维护优先级：

1. 继续压缩首次地图加载体积；
2. 为核心流程补 Playwright 冒烟测试；
3. 清理未接入的旧组件和过期配置；
4. 统一剩余 inline style 到设计 token；
5. 继续检查移动端真实设备体验。

## 六、数据流概览

登录数据流：

1. 用户登录后，后端返回 JWT。
2. 前端将 token 保存到 `localStorage`。
3. 后续 API 请求自动携带 Bearer token。
4. `useStore.load()` 拉取当前用户、访问记录、成就和设置。
5. React 组件订阅 Zustand 状态渲染界面。

访问记录数据流：

1. 用户在城市详情或访问记录表单录入数据。
2. 前端校验城市、天数、日期和备注。
3. 请求后端 `/visits` 创建、更新或删除记录。
4. 后端按当前用户隔离写入 SQLite。
5. 前端更新 `visits` 状态。
6. 地图、统计、排行和成就自动重新计算。

管理员数据流：

1. 管理员打开系统管理。
2. 用户管理读取 `/users` 和 `/stats/system`。
3. 数据管理读取 `/admin/data/export` 得到全站访问记录。
4. 前端按筛选条件展示表格。
5. 生产展示模式不展示批量导入、导出和模板下载入口。

主题数据流：

1. 用户在账户菜单或个人资料中选择主题。
2. 前端调用 `/settings` 保存。
3. 页面根节点更新 `data-theme`。
4. CSS 变量驱动按钮、面板、表格、地图标注和滚动条颜色变化。
