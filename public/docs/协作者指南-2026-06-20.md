# 协作者指南

本文档说明协作者（人或 AI）在修改代码时，如何同步维护项目文档。

---

## 一、文档清单

| 文档 | 路径 | 用途 |
|------|------|------|
| 设计系统文档 | `docs/设计系统-2026-06-20.md` | CSS 变量、组件规范、可访问性规范 |
| 功能现状文档 | `docs/功能现状-2026-06-19.md` | 项目功能清单、技术栈、运行结构 |
| 协作者指南 | `docs/协作者指南-2026-06-20.md` | 本文档，说明文档维护规则 |

## 二、什么时候更新文档

### 必须更新的情况

| 改动类型 | 更新哪个文档 | 更新什么 |
|----------|-------------|----------|
| 新增/修改 CSS 变量 | 设计系统文档 | "基础 Token"章节 |
| 新增/修改组件样式 | 设计系统文档 | "组件规范"章节 |
| 新增可访问性特性 | 设计系统文档 | "可访问性规范"章节 |
| 新增功能 | 功能现状文档 | 对应功能章节 |
| 修改功能逻辑 | 功能现状文档 | 对应功能描述 |
| 删除功能 | 功能现状文档 | 从文档移除 |
| 任何代码改动 | 管理员面板 | CHANGELOG 数组追加记录 |

### 不需要更新的情况

- 纯 bug 修复（不改功能行为）
- 注释、格式调整
- 内部重构（不改对外接口）

## 三、怎么更新文档

### 步骤

1. **找到对应文档** — 见上方文档清单
2. **修改内容** — 按实际代码改动更新
3. **更新头部日期** — `last_updated: YYYY-MM-DD`
4. **追加变更历史** — 在 `changelog` 数组顶部追加一条
5. **git commit** — message 写清楚改了什么

### 变更历史格式

```yaml
changelog:
  - date: 2026-06-21          # 最新日期在前
    author: 协作者名称
    changes:
      - 具体改动1
      - 具体改动2
  - date: 2026-06-20
    author: Hermes Agent
    changes:
      - 之前的改动...
```

## 四、管理员面板升级记录

每次代码改动，在 `src/components/AdminPanel.tsx` 的 `CHANGELOG` 数组**顶部**追加一条：

```typescript
const CHANGELOG = [
  { date: '2026-06-21', items: ['新增XX功能', '修复XX问题'] },  // ← 新的在前
  { date: '2026-06-20', items: ['之前的改动...'] },
];
```

**格式要求：**
- 同一天可以有多条（不同协作者、不同提交）
- 每条 items 是数组，每项是一个改动描述
- 按数组顺序显示（新的在前）

## 五、检查清单

提交代码前，确认：

- [ ] 文档内容与实际代码一致？
- [ ] 文档头部 `last_updated` 日期已更新？
- [ ] 文档 `changelog` 已追加变更条目？
- [ ] 管理员面板 CHANGELOG 已追加记录？
- [ ] git commit message 写清楚了改动内容？

---

## 六、常见问题

**Q: 改了 CSS 变量但忘了更新设计系统文档怎么办？**
A: 下次提交时补上，在 commit message 里注明 `docs: 补充设计系统文档`。

**Q: 功能现状文档太长，更新起来麻烦？**
A: 只更新改动的章节，不需要全文审查。

**Q: 不确定该不该更新文档？**
A: 问一句："这个改动会影响其他协作者的理解吗？" 如果会，就更新。

## 七、文档版本说明

每个文档头部都有 YAML frontmatter：

```yaml
---
title: 文档标题
last_updated: 2026-06-20      # 最后更新日期
maintainer: 维护人名称
related_files:                 # 关联的代码文件
  - src/index.css
changelog:                     # 变更历史（最新在前）
  - date: 2026-06-20
    author: 协作者
    changes:
      - 改动描述
---
```

这些信息帮助协作者快速了解：
- 文档最后什么时候更新的
- 谁在维护
- 关联哪些代码文件
- 最近改了什么
