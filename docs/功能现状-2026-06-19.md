# 城市足迹地图 · 功能与维护指南

> 最后更新：2026-07-20

## 1. 项目定位

城市足迹地图是一份用于保存个人旅行与生活记忆的轻量应用。它以中国城市地图为主视图，用停留天数、最后停留日期和备注记录去过的城市，并通过统计、排行和成就帮助回顾这些经历。

项目面向个人和少量好友，不包含付费、组织协作、复杂审批或商业运营功能。后续扩展应继续以“记录方便、回看愉快、数据可靠”为优先目标。

## 2. 角色与入口

### 普通用户

- 注册并登录自己的账号；
- 浏览全国与省级地图；
- 添加、修改和删除自己的城市记录；
- 查看足迹统计、城市排行和成就；
- 修改昵称、密码和界面主题。

普通用户只能访问自己的记录。

### 管理员

管理员拥有普通用户的全部能力，并可以：

- 创建、修改和删除用户；
- 重置普通用户密码；
- 查看全站访问记录和基础统计；
- 查看功能文档、设计文档和升级记录。

管理员入口位于右上角账号菜单的“系统管理”。

## 3. 核心功能

### 地图

首页默认展示全国城市地图。点亮颜色可按两种方式查看：

- 按停留时间：累计停留越久，颜色越深；
- 按最后离开时间：最近去过的城市更突出。

桌面端支持滚轮缩放和拖拽，移动端支持拖动和双指缩放。左侧工具条用于重置、放大和缩小地图。进入省级视图后，可通过左上角层级按钮返回全国。

### 城市记录

一条记录包含：

- 城市；
- 停留天数；
- 最后停留日期；
- 备注。

同一城市可以保存多条记录，用来区分不同时间的经历。停留天数必须是大于零的整数；日期按设备本地日期处理。

### 足迹统计

足迹统计是项目的核心功能，包含：

- 点亮城市数与城市覆盖率；
- 访问次数；
- 覆盖省份；
- 累计停留天数；
- 成就徽章；
- 停留时长排行。

桌面端统计面板位于地图右侧，移动端位于地图下方。面板可以收起，但不应从公开界面移除。

### 主题

当前支持五套主题：

| 名称 | 代码 | 气质 |
|---|---|---|
| 樱花 | `rose` | 温暖、柔和 |
| 晨雾 | `stripe` | 安静、偏紫 |
| 琥珀 | `amber` | 明亮、偏金 |
| 绿松 | `turquoise` | 清新、偏青 |
| 冰河 | `azure` | 清爽、偏蓝 |

主题会同时影响按钮、表格、弹窗、滚动条和地图颜色。新增界面不能只适配其中一种主题。

## 4. 展示范围

当前公开界面保留：

- 登录与公开注册；
- 地图和城市记录；
- 足迹统计、排行和成就；
- 个人资料与主题切换；
- 管理员基础维护；
- 系统升级记录。

Excel 导入、导出和模板下载仍保留在工程代码中，但生产环境默认隐藏。这些能力是维护工具，不是日常使用入口。

## 5. 数据与隐私

应用使用 SQLite 保存数据，主要包含：

- `users`：账号、昵称、密码哈希和管理员标记；
- `visits`：城市、停留天数、最后停留日期和备注；
- `achievements`：已解锁成就；
- `settings`：用户主题设置。

密码只保存哈希，不能从系统中查看原密码。管理员重置密码时会写入新的哈希。

生产数据库位于服务器，不应提交到代码仓库。备份脚本为：

```bash
./venv/bin/python scripts/backup_cityprint_db.py
```

每次重要交付前应创建一次备份，并使用 SQLite `PRAGMA integrity_check` 验证备份文件。恢复数据库前必须先停止 `cityprint-api.service`。

## 6. 技术结构

- 前端：React、TypeScript、Vite、Zustand、ECharts；
- 后端：FastAPI、SQLite、JWT；
- 样式：CSS 语义变量和项目组件样式；
- 包管理：pnpm；
- 生产地址前缀：`/cityprint/`；
- API 地址前缀：`/cityprint/api/`。

主要目录：

| 目录 | 作用 |
|---|---|
| `src/components` | 页面与业务组件 |
| `src/store` | 登录、用户、访问记录和界面状态 |
| `src/styles` | token、布局、组件和响应式样式 |
| `src/data` | 城市、成就、文档索引和升级记录 |
| `public/geojson` | 地图边界数据 |
| `backend/routers` | API 路由 |
| `backend/services` | 用户、记录、设置和数据服务 |
| `backend/tests` | 后端接口测试 |
| `tests/e2e` | 桌面端与移动端主流程检查 |

## 7. 日常维护

修改后至少执行：

```bash
pnpm run lint
pnpm run build
pnpm run test:e2e
./venv/bin/python -m pytest -q backend/tests
```

发布前还应人工检查：

1. 普通用户和管理员能否登录；
2. 全国地图、省级地图、缩放和重置是否正常；
3. 城市记录能否添加、编辑和删除；
4. 统计面板在桌面端和移动端是否遮挡内容；
5. 五套主题下文字、按钮和表格是否清晰；
6. 个人资料、管理员面板和升级记录是否能正常滚动；
7. 数据库是否已经备份。

## 8. 已确认的产品决定

- 保留公开注册；
- 足迹统计是重要功能，不能为了简化界面而删除；
- 维护型数据工具保留代码，但生产界面默认隐藏；
- 系统文档面向接收人与维护者，不记录内部开发讨论；
- 项目不按 SaaS 设计，不增加付费、复杂权限和运营后台；
- 优先保证数据安全、核心体验和可维护性，再考虑大型重构。
