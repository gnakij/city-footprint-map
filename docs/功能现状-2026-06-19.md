# 城市足迹地图 · 功能现状文档

> 说明：本文档按 2026-06-19 当前仓库代码核实整理，只描述已经接入运行入口的功能，并单独标注"文件存在但未使用"的死代码/残留配置。代码依据主要来自 `src/App.tsx`、`src/store/useStore.ts`、`src/store/api.ts`、`backend/app.py`、各组件文件和 `src/index.css`。

## 一、产品定位

城市足迹地图用于记录登录用户到访过的中国城市、估算停留天数、最后停留日期和备注，并在地图上按城市点亮显示。项目支持多用户登录，访问记录、成就和主题设置按当前登录用户隔离；管理员可以查看和维护全站用户及全站访问记录。

当前主界面不包含分享海报入口。`PosterGenerator` 文件存在但未接入，详见“死代码/未使用功能”。

## 二、技术栈与运行结构

- 前端：React 18 + TypeScript + Vite + Zustand + ECharts + 自定义 CSS。Tailwind 已移除；当前 UI 样式主要写在 `src/index.css`，由 CSS token、自定义 class 和少量 inline style 组成，PostCSS 仅保留 autoprefixer。
- 后端：FastAPI + SQLite + JWT。`backend/app.py` 实际使用 `fastapi`、`jose.jwt`、`passlib`、`slowapi`，不是仅“FastAPI 风格”。
- 前后端 API：前端统一从 `src/store/api.ts` 请求 `/cityprint/api/*`，请求头使用 `Authorization: Bearer <token>`。
- 共享 UI 组件：`src/components/ui` 是指向 `/opt/shared-ui-components/react/src` 的软链接。当前代码实际使用 `DateInput` 和 `FuzzySelect`。
- 部署相关：仓库中只看到 `backend/start.sh`、Vite 构建配置和静态资源；“rsync 到 `/var/www/cityprint/`、systemd service”不在当前仓库代码中体现，本文档不把它当作已核实实现。

## 三、核心数据模型

- `CityData`：前端静态城市库，字段包括 `city_id`、`city_name`、`province`、`region`、`pinyin`、`level`、`adcode`。后端 `/api/cities` 通过解析 `src/data/cities.ts` 暴露城市数据。
- `VisitRecord`：粗粒度访问记录，字段为 `id`、`city_id`、`duration_days`、`last_stay_date`、可选 `notes`、`created_at`、`updated_at`。同一城市允许多条记录，不做时间区间重叠校验。
- `Achievement`：成就定义在 `src/data/achievements.ts`，前端保存访问记录后调用 `checkAchievements` 判断并通过 `/api/achievements` 写入后端。
- `User`：用户字段为 `id`、`username`、`name`、`is_admin`、`created_at`。管理员权限由后端 `admin_user` 依赖保护。
- `AppSettings`：当前只有 `theme`，可选 `rose`、`stripe`、`amber`、`turquoise`、`azure`。

## 四、已接入功能模块

### 4.1 应用入口与加载流程

代码依据：`src/App.tsx`、`src/main.tsx`、`src/store/useStore.ts`。

- 应用启动时调用 `useStore.load()`。
- 如果本地没有 `cityprint-token`，进入登录页。
- 如果有 token，则请求 `/users/me` 拉取当前用户，再并发加载 `/visits`、`/achievements`、`/settings`。
- 管理员登录后额外请求 `/users`，普通用户不会加载用户列表。
- `MapView`、`CityDrawer`、`UserProfile` 使用 React `lazy` 懒加载；应用 hydrated 后会后台预取 `MapView`。
- 已登录后的主界面由 `TopBar`、地图区域、`StatsPanel`、可选 `CityDrawer`、可选 `UserProfile`、`Toast` 组成。

### 4.2 账号、注册与登录

代码依据：`LoginPage.tsx`、`useStore.ts`、`store/api.ts`、`backend/app.py`。

- 登录页有“用户登录”和“管理员登录”两个 tab；用户登录提供用户名、密码输入框和“注册新用户”入口。
- 注册表单包含用户名、昵称、密码、确认密码，前端校验必填、密码至少 6 位、两次密码一致，调用 `/auth/register` 创建普通用户，再调用 `/auth/login` 登录。
- 管理员登录调用 `verifyAdmin`，会先登录再检查 `user.is_admin`，不是管理员则清除 token 并返回失败。
- 后端注册接口 `/api/auth/register` 当前允许 payload 中传 `is_admin`；前端普通注册固定传 `false`，管理员创建用户也传 `false`。首次管理员初始化状态 `adminSetupRequired` 在当前 `load()` 中不会被后端接口驱动，后端数据库为空时会自动创建默认管理员 `admin/admin123456`。
- 登录成功后 token 写入 `localStorage`，后续所有 API 请求自动携带 Bearer token；401 会清除 token 并要求重新登录。

权限：

- 未登录用户只能访问登录/注册 UI。
- 普通用户可注册、登录、修改自己的昵称/密码、管理自己的访问记录。
- 管理员可登录管理入口，并访问后端管理员接口。

### 4.3 顶部栏

代码依据：`TopBar.tsx`。

- 顶部栏展示“城市足迹”和当前用户昵称/用户名。
- 点击用户名打开 `UserProfile` 弹窗。
- “退出”按钮调用 `logout()`，清空 token、当前用户、记录、成就、打开状态，并把主题恢复为 `rose`。

### 4.4 地图主视图

代码依据：`MapView.tsx`、`data/china-geojson.ts`、`utils/colors.ts`、`utils/mapHelpers.ts`。

- 使用 ECharts 渲染地图。全国视图注册 `china-cities-footprint` 市级 GeoJSON 和 `china-provinces-outline` 省界轮廓 GeoJSON。
- 全国视图直接显示城市粒度色块，省界轮廓作为第二个 map series 叠加，只描边、不接收点击。
- 点击全国视图中的普通城市时，先根据 GeoJSON `parent.adcode` 下钻到所属省份；点击直辖市/港澳等特殊行政区时直接打开城市抽屉。
- 省级视图点击城市区域会打开 `CityDrawer`。
- 鼠标 hover 未点亮城市时设置 `previewCity`，地图用高亮色预览；触摸交互期间会屏蔽 hover 预览，避免移动端滑动触发重绘。
- 地图控件包括：
  - “返回省级视图”：仅省级视图出现，实际作用是返回全国视图；
  - “重置视图”：回到全国视图并重置 zoom；
  - “+”“-”：以地图中心放大/缩小；
  - “按停留时间/按最后离开时间”：切换 `colorMode`。
- 自定义交互：
  - 禁用 ECharts 自带 `roam`；
  - 桌面端滚轮缩放、鼠标拖拽平移；
  - 移动端单指平移使用 rAF 合并 touchmove；
  - 双指缩放期间先用 CSS transform 预览，松手后一次性同步到 ECharts；
  - 全国视图下主图层和省界图层同时 dispatch `geoRoam`，保证缩放/平移同步。
- 数据流：
  - 地图只读前端 Zustand 中的 `visits`、`previewCity`、`colorMode`；
  - 颜色按访问记录聚合计算，不直接请求后端；
  - 选择城市后写入 `selectedCity` 并打开抽屉。

权限：已登录普通用户和管理员都可以使用地图；地图展示的是当前登录用户自己的访问记录。

### 4.5 城市详情与记录管理

代码依据：`CityDrawer.tsx`、`useStore.ts`、`store/api.ts`、`backend/app.py`。

- 抽屉头部有“返回”和“×”关闭按钮。
- 展示城市名、省份、大区，以及“共 N 天 · M 条记录”或“未记录”状态。
- 已有记录以卡片展示：停留天数、最后停留日期、备注，并提供“编辑”“删除”按钮。
- “添加访问”展开表单，字段包括：
  - 停留天数：数字输入，最小值 1；
  - 最后停留日期：共享组件 `DateInput`；
  - 备注：textarea，可选。
- 编辑记录会复用同一表单并填入原值，提交按钮变成“保存修改”。
- 保存时调用 Zustand `saveVisit`：
  - 前端校验登录、日期、天数；
  - 构造 `VisitRecord`；
  - `store/api.ts` 先尝试 `PUT /visits/{id}`，如果 404 再 `POST /visits`；
  - 保存成功后重新计算成就，必要时调用 `/achievements` 写入解锁记录。
- 删除调用 `DELETE /visits/{visit_id}`，后端校验记录必须属于当前用户。

权限：普通用户和管理员都只能在此管理自己当前登录账号的访问记录，不能在抽屉里改其他用户的数据。

### 4.6 统计面板与下钻统计

代码依据：`StatsPanel.tsx`、`DrillDownStats.tsx`、`useStore.ts`。

- 右下角 `StatsPanel` 展示：
  - 点亮城市数/总城市数；
  - 访问次数；
  - 覆盖省份；
  - 累计天数；
  - 城市覆盖率进度条；
  - 成就徽章预览；
  - 城市排行。
- 四个统计数字都是按钮，点击会打开 `UserProfile` 的“访问记录”tab。
- 面板右上角“▼”可收起；收起后显示浮动 pill，点击展开。
- 成就默认显示前 8 个，可点击“展开全部/收起”。
- 排行默认显示前 10 个，可点击“查看全部/收起”；当 `colorMode=duration` 时按累计天数排序，当 `colorMode=lastDeparture` 时按最后停留日期排序。
- `DrillDownStats` 当前通过 `UserProfile` 的“统计”按钮嵌入访问记录 tab：
  - 可在“按天数/按名称”之间切换排序；
  - 第一级按省份聚合；
  - 点击省份进入城市列表；
  - 点击城市进入该城市记录列表；
  - “返回”逐级回退。
- `DrillDownStats` 也保留非嵌入弹窗模式，但当前主入口没有直接打开独立统计弹窗。

权限：只读取当前登录用户的前端访问记录状态。

### 4.7 成就系统

代码依据：`achievements.ts`、`AchievementBadge.tsx`、`StatsPanel.tsx`、`useStore.ts`。

- 当前共有 10 个成就：
  - 初次点亮；
  - 十城达人；
  - 省份征服者；
  - 百城俱乐部；
  - 长居达人；
  - 神州北极；
  - 东极迎光；
  - 南海逐浪；
  - 西域极境；
  - 全国制霸。
- 解锁条件在前端函数中计算，依据当前访问记录和静态城市库。
- 保存单条或批量访问记录后触发成就检查；新解锁时写入后端 `/api/achievements` 并弹 toast。
- 成就徽章在统计面板中展示，未解锁/已解锁由 `achievements` id 列表决定。

权限：成就按用户隔离，普通用户和管理员都只看当前账号自己的成就。

### 4.8 用户资料、访问记录、Excel 导入导出、设置

代码依据：`UserProfile.tsx`。

`UserProfile` 是当前个人中心的实际入口，顶部通过 tab 切换：

- “个人信息”：
  - 展示只读用户名；
  - 展示昵称；
  - “修改昵称”进入编辑状态，提供“保存/取消”；
  - “修改密码”展开新密码和确认密码表单，提供“更新密码/取消”；
  - 底部展示用户类型和创建日期。
- “访问记录”：
  - 表格列为省份、城市、天数、最后停留、备注、操作；
  - 每行可“编辑”或“删除”；
  - 操作区包括“添加访问”“导入数据”“导出数据”“下载模板”“清空所有数据”“统计”。
- 添加/编辑访问记录：
  - 城市选择使用共享 `FuzzySelect`，支持城市名和拼音；
  - 停留天数数字输入；
  - 最后停留日期使用 `DateInput`；
  - 备注输入框；
  - 提供“添加记录/保存修改”和“取消”。
- Excel 导出：
  - 使用 `xlsx` 生成 `城市足迹备份.xlsx`；
  - 导出列为省份、城市、停留天数、最后停留日期、备注。
- Excel 导入：
  - 文件类型为 `.xlsx`；
  - 读取第一个 sheet；
  - 支持 Excel 日期单元格和文本日期；
  - 校验城市必填、城市匹配、停留天数为正整数、日期格式、当前用户已有城市重复；
  - 展示导入预览，区分有效、跳过、错误；
  - “确认导入”会调用 `bulkCreateVisits`。
- 注意：普通用户 `bulkCreateVisits` 实际是前端 `Promise.all` 并发调用多次 `POST /visits`，不是后端批量接口。
- “清空所有数据”需要浏览器 `confirm`，确认后调用 `/data/clear` 清空当前用户访问记录和成就。
- “统计”切换到内嵌 `DrillDownStats`，可返回访问列表。
- “系统设置”：
  - 当前只提供主题下拉框，选项为 `rose`、`stripe`、`amber`、`turquoise`、`azure`；
  - 保存后调用 `/settings`，并更新 `document.documentElement.dataset.theme`。

权限：

- 普通用户可看到“个人信息”“访问记录”“系统设置”。
- 管理员会额外看到“系统管理”tab。
- 用户资料中的访问记录和设置只作用于当前登录用户。

### 4.9 管理后台

代码依据：`AdminPanel.tsx`、`UserProfile.tsx`、`store/api.ts`、`backend/app.py`。

入口：

- 当前管理后台只通过管理员用户打开 `UserProfile` 后的"系统管理"tab 进入。
- `AdminPanel` 使用 `lazy(() => import('./AdminPanel'))`，普通用户不会看到该 tab。

管理后台有三个页签：

- **用户管理**：用户列表、新增用户、重置密码、删除用户
- **数据管理**：全站访问记录查看、筛选、导出、导入
- **系统文档**：集中展示设计文档、功能文档、协作者指南，以及系统升级记录（点击链接弹窗展示）

用户管理 tab：

- 顶部显示系统统计：总用户、管理员、总访问，来自 `/stats/system`。
- "新增用户"打开弹窗，字段为用户名、昵称、密码，创建普通用户。
- 用户表格列为用户、类型、创建时间、密码、操作。
- 用户昵称可在表格 input 内编辑，失焦后调用 `/users/{id}` 更新。
- "重置密码"打开弹窗，包含新密码、确认密码，"确认重置/取消"，密码至少 6 位且两次一致。
- 非管理员用户显示"删除"按钮；管理员用户当前 UI 不显示删除按钮。后端也禁止删除自己，并保证至少保留一个管理员。

数据管理 tab：

- 切到该 tab 时调用 `/admin/data/export` 拉取所有用户访问记录到前端状态 `allVisits`。
- 筛选区包含 4 个 `FuzzySelect`：
  - 用户名；
  - 昵称；
  - 省份；
  - 城市。
- 用户名和昵称使用 `pinyin-pro` 动态生成拼音 key；省份使用固定拼音字典；城市使用 `CITIES` 的 `pinyin` 字段。
- "查询"按钮把当前输入写入 `appliedFilters`，表格才按筛选条件更新。
- "导出当前视图"导出当前过滤后的 ledger 为 `城市足迹当前视图.xlsx`。
- "导入数据"展开导入工具：
  - 先选择目标用户；
  - "选择文件"读取 `.xlsx`；
  - 导入前会请求目标用户现有记录，校验城市匹配、停留天数、日期、目标用户已有城市、文件内重复；
  - 展示导入预览；
  - "确认导入"调用 `/admin/data/import`，这是后端批量插入接口。
- "下载模板"导出管理员导入模板。
- 数据表格分页，每页 10 条，有"上一页/下一页"。

系统文档 tab：

- 展示项目文档列表，包括：
  - 设计系统文档（链接到 markdown 文件）
  - 功能现状文档（链接到 markdown 文件）
  - 协作者指南（链接到 markdown 文件）
  - 系统升级记录（点击链接弹窗展示 CHANGELOG 数组）

权限：

- 前端入口只对 `currentUser.is_admin` 显示。
- 后端 `/api/users`、`/api/users/{id}`、`/api/admin/data/export`、`/api/admin/data/import`、`/api/stats/system` 均依赖 `admin_user`，普通用户请求会返回 403。

### 4.10 全局提示与错误边界

代码依据：`Toast.tsx`、`ErrorBoundary.tsx`。

- `Toast` 读取 Zustand `toast`，展示 icon 和 message，3 秒后自动隐藏。
- `ErrorBoundary` 包裹整个 App，捕获渲染错误并打印到 console。

### 4.11 图标系统

代码依据：`Icon.tsx`。

项目使用统一的 SVG 图标组件 `Icon.tsx`，替代原有的 emoji 和文字符号图标。

**可用图标**：

| 名称 | 用途 | 替换前 |
|------|------|--------|
| `close` | 关闭按钮 | × |
| `plus` | 添加/放大 | + |
| `minus` | 缩小 | − |
| `refresh` | 重置视图 | 🔄 |
| `download` | 下载/导入 | 📥 |
| `upload` | 上传/导出 | 📤 |
| `chart` | 统计 | 📊 |
| `search` | 查询 | 🔍 |
| `key` | 管理员标识 | 🔑 |
| `user` | 用户标识 | 👤 |
| `city` | 城市图标 | 🏙️ |
| `check` | 成功/确认 | ✓/✅ |
| `warning` | 警告 | ⚠️ |
| `error` | 错误 | ❌ |
| `info` | 信息/提示 | ! |

**使用方式**：

```tsx
import Icon from './Icon';

<Icon name="close" />

```

**保留的 emoji**（品牌/装饰元素，不替换）：

## 八、新人理解用数据流总览

登录态数据流：

1. 登录/注册成功后，后端返回 JWT。
2. 前端把 JWT 存入 `localStorage` 的 `cityprint-token`。
3. `request()` 为后续 API 自动加 Bearer token。
4. `useStore.load()` 根据 token 拉取当前用户、访问记录、成就、设置。
5. Zustand 保存前端运行时状态，React 组件订阅状态渲染。

访问记录数据流：

1. 用户在地图抽屉或个人中心表单录入城市、停留天数、最后停留日期、备注。
2. 前端校验后调用 `saveVisit`。
3. `saveVisit` 经 `store/api.ts` 请求后端 `/visits`。
4. 后端写入 SQLite `visits` 表，并只允许访问当前 token 所属用户的数据。
5. 前端更新 Zustand `visits`。
6. 地图、统计、排行、下钻统计自动基于新的 `visits` 重新计算。
7. 前端检查成就，必要时调用 `/achievements` 持久化。

管理员数据流：

1. 管理员进入 `UserProfile` 的“系统管理”tab。
2. 用户管理读取 `/users` 和 `/stats/system`。
3. 数据管理调用 `/admin/data/export` 一次性拉取用户范围内的访问记录到前端。
4. 筛选、分页、导出当前视图主要在前端完成。
5. 管理员 Excel 导入调用 `/admin/data/import` 批量写入目标用户。

主题数据流：

1. 用户在个人中心“系统设置”选择主题。
2. 前端调用 `/settings` 保存到 SQLite。
3. 前端同步更新 `document.documentElement.dataset.theme`。
4. `src/index.css` 根据 `:root[data-theme="..."]` 切换 CSS 变量。

权限边界：

- 未登录：只能登录/注册。
- 普通用户：地图、自己的访问记录、自己的成就、自己的主题设置、自己的资料。
- 管理员：拥有普通用户能力；额外拥有用户列表、用户昵称修改、重置密码、删除普通用户、全站访问记录查看、按目标用户导入记录、系统统计。
- 后端强制保护管理员接口；普通访问记录接口始终按当前 token 用户隔离。
