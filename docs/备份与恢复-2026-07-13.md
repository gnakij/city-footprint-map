# 城市足迹备份与恢复

本文档记录 SQLite 数据库的服务器侧备份和恢复流程。

## 备份脚本

脚本位置：

```bash
scripts/backup_cityprint_db.py
```

默认行为：

- 数据库路径：`CITYPRINT_DB_PATH`，未设置时使用 `backend/data/cityprint.db`
- 备份目录：`CITYPRINT_BACKUP_DIR`，未设置时使用 `backend/backups`
- 保留时间：`CITYPRINT_BACKUP_RETENTION_DAYS`，未设置时保留 30 天
- 备份文件名：`cityprint-YYYYMMDDTHHMMSSZ.db`

手动执行：

```bash
cd /opt/city-footprint-map
python3 scripts/backup_cityprint_db.py
```

指定路径执行：

```bash
python3 scripts/backup_cityprint_db.py \
  --db /opt/city-footprint-map/backend/data/cityprint.db \
  --backup-dir /opt/city-footprint-map/backend/backups \
  --retention-days 30
```

## 定时备份

推荐用 cron 每天执行一次：

```cron
17 3 * * * cd /opt/city-footprint-map && /usr/bin/python3 scripts/backup_cityprint_db.py >> /var/log/cityprint-backup.log 2>&1
```

如果部署时数据库放在其他路径，显式传参或设置环境变量：

```cron
17 3 * * * cd /opt/city-footprint-map && CITYPRINT_DB_PATH=/data/cityprint.db CITYPRINT_BACKUP_DIR=/data/cityprint-backups /usr/bin/python3 scripts/backup_cityprint_db.py >> /var/log/cityprint-backup.log 2>&1
```

## 恢复流程

恢复前必须先停止后端服务，避免运行中的进程继续写入旧数据库。

```bash
sudo systemctl stop cityprint
```

先保留当前数据库：

```bash
cd /opt/city-footprint-map
cp backend/data/cityprint.db backend/data/cityprint.db.before-restore.$(date -u +%Y%m%dT%H%M%SZ)
```

复制目标备份：

```bash
cp backend/backups/cityprint-YYYYMMDDTHHMMSSZ.db backend/data/cityprint.db
rm -f backend/data/cityprint.db-shm backend/data/cityprint.db-wal
```

启动服务并检查：

```bash
sudo systemctl start cityprint
sudo systemctl status cityprint --no-pager
```

如果当前部署不是 systemd 服务，按实际进程管理方式停止和启动后端即可，关键点是恢复文件时不能让后端同时写库。
