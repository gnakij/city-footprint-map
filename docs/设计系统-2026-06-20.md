# 城市足迹地图 · 设计系统

> 关联文件：`src/index.css`（token 定义）、`CLAUDE.md`（UI 规范）

---

## 一、设计理念

### 1.1 品牌定位

| 维度 | 定义 |
|------|------|
| **目标用户** | 旅行爱好者，记录去过的城市 |
| **情感调性** | 温馨旅行日记感 |
| **视觉隐喻** | 日记本、邮票、地图 |
| **设计原则** | ①温馨不花哨 ②地图边框硬编码不动 ③字重不改 ④页签方形非胶囊 |

### 1.2 设计原则（决策准则）

1. **温馨优先**：圆角、柔和阴影、暖色调，避免冷硬直角
2. **一致性**：所有组件引用 design token，不硬编码数值
3. **可维护**：变量化 > 硬编码，改一处全局生效
4. **克制**：不随意添加功能，新增内容融入现有设计

## 四、地图规范（特殊）

### 4.1 硬编码规则

以下属性**硬编码，不跟随主题**：

| 属性 | 值 | 原因 |
|------|------|------|
| 省界轮廓 `borderColor` | `#666666` | 移动端反馈"太粗发黑"，从 `#B98A98` 调整 |
| 省界轮廓 `borderWidth` | `0.5` | 细灰色描边 |
| 地图标签 `fontWeight` | 不改 | 用户偏好细体 |

### 4.2 性能优化

- `devicePixelRatio` 钳到 2，避免高清屏过度渲染
- 双指 pinch 缩放+平移走 CSS transform（GPU 合成）
- 触摸期间屏蔽 hover 预览（`isTouchingRef` 标志位）

## 五、可访问性规范

### 5.1 Focus Ring

所有可交互元素必须有 `:focus-visible` 样式：

```css
:focus-visible {
  outline: 2px solid var(--color-focus-ring);
  outline-offset: 2px;
}
```

### 5.2 颜色对比度

遵循 WCAG AA 标准：
- 正文文字：对比度 ≥ 4.5:1
- 大标题（≥18px 或 ≥14px bold）：对比度 ≥ 3:1

**对比度检查结果**：

| 颜色组合 | Rose | Linear | Stripe | 说明 |
|----------|------|--------|--------|------|
| 主要文字 on 表面 | ✅ 17.40:1 | ✅ 17.40:1 | ✅ 17.40:1 | |
| 次要文字 on 表面 | ✅ 5.74:1 | ✅ 5.74:1 | ✅ 5.74:1 | |
| 主色 on 背景 | ✅ 4.07:1 | ✅ 4.42:1 | ✅ 4.42:1 | 大标题用途 |
| 白色 on 主色 | ⚠️ 4.35:1 | ✅ 4.70:1 | ✅ 4.70:1 | Rose 按钮文字，差 0.15 |
| 成功色 on 表面 | ⚠️ 4.12:1 | — | — | 仅用于图标/边框，不用于正文 |
| 警告色 on 表面 | ⚠️ 2.70:1 | — | — | 仅用于图标/边框，不用于正文 |
| 错误色 on 表面 | ✅ 4.98:1 | — | — | |

**使用建议**：
- Rose 主题按钮文字（白色 on 主色）对比度略低，但实际使用中可读性良好
- 成功色/警告色主要用于图标、边框、小面积提示，避免用于大段正文
- Linear/Stripe 主题对比度全部达标

### 5.3 减少动效

尊重用户系统设置：

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 5.4 触摸目标

移动端可点击区域最小尺寸：
- Apple HIG: 44x44px
- Material Design: 48x48px

当前按钮高度 36px，需确保 padding 或 margin 补足到 44px 触摸区域。

## 六、响应式断点

项目使用三个断点（CSS @media 不支持变量引用，断点值直接写在媒体查询中）：

| 名称 | 数值 | 用途 | 媒体查询 |
|------|------|------|----------|
| 移动端 | ≤767px | 手机布局 | `@media (max-width: 767px)` |
| 桌面端 | ≥768px | 桌面布局 | `@media (min-width: 768px)` |
| 平板 | 768-1024px | 平板适配 | `@media (min-width: 768px) and (max-width: 1024px)` |

**响应式策略**：
- Mobile First：先写移动端，用 `min-width` 媒体查询增强
- 流式布局：`flex` / `grid` + `clamp()` + 相对单位
- 内容优先级：移动端隐藏次要内容

## 七、全局状态规范

项目定义了三种全局状态类名，所有页面统一使用：

### 7.1 空状态

```css
.empty-state {
  display: grid;
  place-items: center;
  gap: var(--space-3);
  padding: var(--space-6);
  color: var(--color-on-surface-variant);
  text-align: center;
}
```

### 7.2 加载态

```css
.loading {
  display: grid;
  place-items: center;
  padding: var(--space-6);
}

.loading-spinner {
  width: 32px;
  height: 32px;
  border: 3px solid var(--color-outline-variant);
  border-top-color: var(--color-primary);
  border-radius: 50%;
  animation: spin 0.8s linear infinite;
}
```

### 7.3 错误态

```css
.error-state {
  display: grid;
  gap: var(--space-2);
  padding: var(--space-4);
  background: rgba(211, 47, 47, 0.08);
  border: 1px solid var(--color-error);
  border-radius: var(--radius-panel);
  color: var(--color-error);
}
```
