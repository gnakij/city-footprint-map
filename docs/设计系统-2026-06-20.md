## 一、设计理念

### 1.1 品牌定位

| 维度 | 定义 |
|------|------|
| **目标用户** | 旅行爱好者，记录去过的城市 |
| **情感调性** | 温馨旅行日记感 |
| **视觉隐喻** | 日记本、邮票、地图 |
| **设计原则** | ①温馨不花哨 ②地图边框硬编码不动 ③字重不改 ④页签方形非胶囊 |

### 1.2 设计原则（决策准则）

1. **温馨优先**：圆角、柔和阴影、暖色调，避免冷硬直角
2. **一致性**：所有组件引用 design token，不硬编码数值
3. **可维护**：变量化 > 硬编码，改一处全局生效
4. **克制**：不随意添加功能，新增内容融入现有设计

## 四、地图规范（特殊）

### 4.1 硬编码规则

以下属性**硬编码，不跟随主题**：

| 属性 | 值 | 原因 |
|------|------|------|
| 省界轮廓 `borderColor` | `#666666` | 移动端反馈"太粗发黑"，从 `#B98A98` 调整 |
| 省界轮廓 `borderWidth` | `0.5` | 细灰色描边 |
| 地图标签 `fontWeight` | 不改 | 用户偏好细体 |

### 4.2 性能优化

- `devicePixelRatio` 钳到 2，避免高清屏过度渲染
- 双指 pinch 缩放+平移走 CSS transform（GPU 合成）
- 触摸期间屏蔽 hover 预览（`isTouchingRef` 标志位）

## 六、可访问性规范（新增）

### 6.1 Focus Ring

所有可交互元素必须有 `:focus-visible` 样式：

```css
:focus-visible {
  outline: 2px solid var(--color-focus-ring);
  outline-offset: 2px;
}
```

### 6.2 颜色对比度

遵循 WCAG AA 标准：
- 正文文字：对比度 ≥ 4.5:1
- 大标题（≥18px 或 ≥14px bold）：对比度 ≥ 3:1

### 6.3 减少动效

尊重用户系统设置：

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 6.4 触摸目标

移动端可点击区域最小尺寸：
- Apple HIG: 44x44px
- Material Design: 48x48px

当前按钮高度 36px，需确保 padding 或 margin 补足到 44px 触摸区域。

## 七、响应式断点（待定义）

建议断点：

| 名称 | 数值 | 用途 |
|------|------|------|
| `--breakpoint-mobile` | 0-767px | 手机 |
| `--breakpoint-tablet` | 768-1023px | 平板 |
| `--breakpoint-desktop` | 1024px+ | 桌面 |

**响应式策略**：
- Mobile First：先写移动端，用 `min-width` 媒体查询增强
- 流式布局：`flex` / `grid` + `clamp()` + 相对单位
- 内容优先级：移动端隐藏次要内容
